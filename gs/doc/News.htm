Version 6.21 (2000-xx-xx)

This fileset includes more PDF writer enhancements: conversion of embedded Type 1 fonts to Type 1C (smaller), handling of more color spaces for images, compression of mask images, partial support for (tiled) Pattern colors, more graphics state distiller parameters, recording of graphics state functions in the output. It also greatly improves the space usage and loading time for CMaps.

All problems registered on SourceForge numbered 104316 or lower are closed, except for the following: 101549, 101814, 101829, 101905, 101955, 102046, 102047, 102132, 102146, 102735, 103016, 103528, 104118, 104316.

The following projects are in a partially completed state in the PDF writer:

  • Pattern colors (gdevpdf[ci].c) -- working except for patterns that require masking.
  • CID-keyed fonts (gdevpdf[eftw].c) -- barely started.
  • ImageType 3 images (gdevpdf[ij].c) -- started. 


Incompatible changes


(Drivers)
	- The downsampling filters now require WidthIn and HeightIn,
replacing Columns.
	- The gp_ procedures for managing screen swapping between console
and graphics are removed.
(PDF writer)
	- The last argument of pdf_alloc_font has changed.
	- pdf_set_color is renamed pdf_set_pure_color.
(Interpreter)
	- The .fontproperties procedure in gs_fonts.ps now takes an
additional argument.
(Library)
	- The internal representation of CMaps is changed completely, and
has several new required members.
	- gs_cmap is renamed gs_cmap_t; gx_code_map is renamed
gx_code_map_t.
	- Clients creating CMaps must call gs_cmap_init to initialize them.
