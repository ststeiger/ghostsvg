GSview Common Problems

The two most common problems with GSview 1.4 and earlier are

Can't run gswin or Can't run gsos2
GSview requires a suitable version of Ghostscript (gsos2.exe for OS/2, gswin.exe or gswin32.exe for MS-Windows). This error message occurs if you don't have Ghostscript, or if GSview can't find Ghostscript. From the GSview menu select Options | Ghostscript Command and enter the correct executable path and include path for Ghostscript. For example: 
c:\gs\gswin.exe -Ic:\gs;c:\gs\fonts;c:\psfonts
You must set the include path for Ghostscript using either the -I command or the GS_LIB environment variable.

Ghostscript starts and then immediately exits with error code 1
First read the error message in the Ghostscript window. Ghostscript probably couldn't find its initialisation files. Set the Ghostscript include path by adding -Idirectory to the command in Options | Ghostscript Command, where directory includes the name of the directory that contains the Ghostscript initialisation files,
Other common problems are described in the GSview on-line help. This is available for OS/2 and MS-Windows.
