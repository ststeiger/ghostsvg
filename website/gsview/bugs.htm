(2002-01-25)

Problem: (4.1) Attempting to display any file results in an Unrecoverable error: invalidaccess in .setdevice. This occurs with GSview 4.0 or 4.1 used in conjunction with AFPL Ghostscript 7.04, due to some security improvements.

Workaround: Use GSview 4.12 beta.

Workaround: Disable "Options | Safer".

(2001-11-25)

Problem: (4.1) "gsview /E filename" doesn't work.

Workaround: Use GSview 4.0.

(2001-11-23)

Problem: (4.1) GSview Linux in German shows some Spanish text.

Workaround: Look at src/gvxinit.c lines 858-859. Fix the typo and recompile.

(2001-06-10)

Problem: (4.0) DOS EPS file is invalid if adding TIFF preview and EPS omits trailer. If an EPS file omits the trailer and a TIFF preview is added by GSview or epstool, the DOS EPS file will contain an incorrect size for the PostScript section. The same bug occurs if a user supplied preview is added. This does not affect adding a Windows MetaFile preview.

Workaround: Append the following line to the EPS file before adding the preview 

 %%Trailer

Workaround: Use GSview 4.1.
